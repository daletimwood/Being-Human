05 — Logging

How to add a session score

At the end of any Claude session, paste this prompt to generate a score entry. Claude will assess the session against all 20 active criteria and output a JSON block ready to append to scores.json.

Score this session against the Being Human Framework v1.0. For each of the 20 active criteria (HA-1 through HA-17, EU-1 through EU-3, CC-4), assign a score from 0–10 based on observable prompt signals. Then check EU-4 (pass/fail) and EU-4a (pass/flagged/disqualified). Output a valid JSON object matching the scores.json session schema. Do not include prompt text. Declared session intent: [paste your session declaration here].

Then open scores.json in the GitHub web editor and append the returned object to the "sessions" array. The dashboard fetches live from the raw GitHub URL — no build step required.

Session schema

{
  "session_id": "uuid-v4",
  "declared_at": "2026-06-28T09:00:00Z",
  "session_type": "analytical | creative | administrative | learning | planning | other",
  "intent_summary": "short public description of declared purpose",
  "commitment_hash": "sha256(intent + declared_at)",
  "eu4_status": "PASS | FAIL",
  "eu4a_status": "PASS | FLAGGED | DISQUALIFIED",
  "composite_score": 72.5,
  "layer_scores": {
    "HA": 68.0,
    "EU": 81.5,
    "CC": 75.0
  },
  "criterion_scores": {
    "HA-1": 7, "HA-2": 6, "HA-3": 8, "HA-4": 5,
    "HA-5": 7, "HA-6": 6, "HA-7": 6, "HA-8": 8,
    "HA-9": 7, "HA-10": 5, "HA-12": 6, "HA-13": 7,
    "HA-14": null, "HA-15": 5, "HA-16": 6, "HA-17": 4,
    "EU-1": 8, "EU-2": 7, "EU-3": 9,
    "CC-4": 7
  },
  "prompt_count": 12,
  "context_gap_count": 1,
  "notes": "optional assessor notes — public"
}

HA-11 is reserved for v2. HA-14 is null when no named other is present in the session. Both are excluded from their layer denominator when null.
